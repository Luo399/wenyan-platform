## ** 测试目的**

验证基于 Web 的“Excel 转 JSON 数据展示平台”在 Windows 7 及以上操作系统的主流浏览器中，功能完整、界面正常、无脚本错误，核心组件（如 Wordlist）能正确校验双 JSON 的 `text_id` 及文件名。

## **2. 测试范围**

- **核心功能**：
  - Excel 文件上传 → 转换 JSON → Wordlist 组件加载展示
  - Wordlist 组件双 JSON 一致性校验（文件名、`text_id`）
  - 空值渲染为 `null` 的正确展示
- **UI 表现**：布局、字体、弹窗、错误提示在不同浏览器下的一致性
- **异常处理**：网络失败、JSON 格式错误、校验不通过的提示

## **8. 测试执行建议**

1. **虚拟环境优先**：使用 Hyper-V/VMware 搭建 Win7/Win10 虚拟机，安装各浏览器进行测试。
2. **国产浏览器模式切换**：测试时需手动切换 360/QQ 浏览器的“极速模式”和“兼容模式”，确保两种内核均执行核心用例。
3. **自动化辅助**：可使用 Playwright 编写跨浏览器 E2E 脚本，但 IE 及兼容模式可能需要手动验证。
4. **回归重点**：每次发版前，至少执行 TC-EXCEL-001 和 TC-WORDLIST-001/002 作为冒烟测试。

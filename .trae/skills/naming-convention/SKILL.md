# 文言文平台素材命名规范

## 核心公式

{课文编号}{功能标识}{内容简述}.{后缀}

text

- **课文编号**：`WEN_` + 两位数字，如 `WEN_01`、`WEN_12`（不足两位补零）
- **功能标识**：见下表，固定缩写，不可修改
- **内容简述**：英文小写单词，可选（如 `guqin`、`narrator`）
- **后缀**：`.mp3`（音频）、`.jpg`/`.png`（图片）、`.mp4`（视频）

**通用素材**（不区分课文）：
common\_{功能标识}\_{内容简述}.{后缀}

text

## 开发对接规则

### 资源路径

- 音频：`/audio/{规范文件名}`
- 图片：`/images/{规范文件名}`
- 视频：`/video/{规范文件名}`

### 前端解析逻辑

文件名自带完整信息，无需额外映射表。示例：

```typescript
// 根据课文编号和功能直接拼接
const audioUrl = `/audio/WEN_01_read_full.mp3`
const imageUrl = `/images/WEN_01_illus_room.jpg`
JSON 数据中的使用
在 annotated.json 或 segments.json 中，音频/图片字段直接存储完整文件名（不含路径），前端统一拼接路径：

json
{
  "audioUrl": "WEN_01_read_full.mp3",
  "imageUrl": "WEN_01_illus_room.jpg"
}
渲染时：

vue
<img :src="`/images/${data.imageUrl}`" />
<audio :src="`/audio/${data.audioUrl}`" />
命名冲突处理规则
当 AI 遇到以下情况时，必须暂停并询问用户：
- 同一功能模块下，同一课文有多个同名 JSON 文件 → 询问保留哪个或合并。
- 两个不同模块需要使用同一个原始素材文件（如 illus 和 card 都引用了同一张图） → 询问是复制一份还是共享引用。
- 通用素材（common_xxx）与课文素材文件名完全一致 → 询问使用哪个优先级。
- 生成新 JSON 时，目标文件已存在 → 询问是否覆盖、合并或跳过。

禁止事项
 不要为同一课文的同一功能创建多个 JSON 文件
 不要在 JSON 中存储完整路径（如 /audio/WEN_01_read_full.mp3），只存文件名
 不要忽略命名冲突，必须询问用户
```

### 命名冲突处理规则

当 AI 遇到以下情况时，必须暂停并询问用户：

- 同一功能模块下，同一课文有多个同名 JSON 文件 → 询问保留哪个或合并。
- 两个不同模块需要使用同一个原始素材文件（如 illus 和 card 都引用了同一张图） → 询问是复制一份还是共享引用。
- 通用素材（common\_xxx）与课文素材文件名完全一致 → 询问使用哪个优先级。
- 生成新 JSON 时，目标文件已存在 → 询问是否覆盖、合并或跳过。

## 禁止事项

&#x20;不要为同一课文的同一功能创建多个 JSON 文件

&#x20;不要在 JSON 中存储完整路径（如 /audio/WEN\_01\_read\_full.mp3），只存文件名

&#x20;不要忽略命名冲突，必须询问用户

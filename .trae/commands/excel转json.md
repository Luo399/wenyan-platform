1. **请使用 Excel MCP 读取项目中的 Excel 文件，并按以下要求生成 JSON 数据和配套组件：**
   ***
   **1. 读取与映射规则**
   - 将 Excel 中的**指定表/命名区域**视为一个独立的组件数据源，**表名直接作为组件名**（保持 PascalCase，无空格和特殊符号）。
   - 将每个表的**首行作为属性名**，**后续每一行作为一个数据对象**，生成如下格式：

     json
     ```
     {
       "组件名": [
         { "列1属性": "值", "列2属性": "值" },
         { "列1属性": "值", "列2属性": "值" }
       ],
       ...
     }
     ```
   - **必须用 **`JSON.stringify()`** 序列化整个对象**再写入文件，严禁手动拼接字符串，以避免双引号等特殊字符未转义导致 JSON 解析失败。
   **2. 健壮性要求**
   - 若某个单元格内容为空，保留该属性，值设为空字符串 `""`。
   - 若读取或解析过程中出现任何异常，必须在控制台输出详细错误，并为该表生成空数组 `[]`，不能中断整个生成流程。
   - 为 Excel 解析添加 15 秒超时，超时后返回错误提示并停止后续文件生成。
   **3. 文件输出与变更记录**
   - 将生成的 JSON 文件命名为 `componentData.json`，存放到 `src/data/` 目录（目录不存在则自动创建）。
   - 在项目根目录的 `CHANGELOG.md` 顶部追加一条变更摘要，格式：`## [日期] 从 Excel 生成组件数据`，并列出所有表名和生成的文件。
   - 使用 GitHub MCP 将 `componentData.json` 和 `CHANGELOG.md` 提交到一个新分支 `feature/excel-to-json-日期`，并创建 Pull Request。
   **4. 配套前端优化（防止加载卡死）**
   - 为使用 `componentData.json` 的组件添加：
     - 10 秒加载超时，超时显示“加载超时，请重试”及重试按钮。
     - 错误自动重试一次，并缓存上次成功的数据优先展示。
     - 确保所有异步分支（成功、失败、超时）都正确关闭 loading 状态，避免页面一直转圈。
   **请严格按照以上步骤执行，先完整读取 Excel 并确认表名和列名，再一次性生成 JSON 和配套组件**

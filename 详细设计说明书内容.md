# 详细设计说明书

## 1 引言

### 1.1 编写目的

本详细设计说明书旨在对文言文学习平台（wenyan-platform）系统的各个程序模块进行详细设计说明，明确每个模块的功能、算法、流程逻辑、接口设计等，为编码实现提供直接依据。

预期读者包括：前端开发人员、后端开发人员、测试人员。

### 1.2 背景

- **待开发软件系统的名称**：文言文学习平台（wenyan-platform）
- **项目任务提出者**：教学研究团队
- **开发者**：项目开发团队
- **用户**：中小学学生、语文教师
- **运行环境**：支持现代浏览器的PC端及移动端设备

本系统是一个面向文言文教学的在线学习平台，采用前后端分离架构，前端基于Vue 3 + TypeScript + Vite，后端基于Node.js + Express + SQLite。

## 3 前端核心模块设计说明

### 3.1 路由模块（router/index.ts）

#### 3.1.1 程序描述

路由模块是前端应用的导航核心，负责定义应用的路由规则、页面组件映射、导航守卫等。基于Vue Router 5实现，支持动态路由、嵌套路由、路由守卫等功能。

#### 3.1.2 功能

1. 定义应用路由表，映射URL路径到页面组件
2. 实现路由导航守卫，控制页面访问权限
3. 支持动态路由参数传递
4. 提供编程式导航API
5. 支持404页面匹配

#### 3.1.3 性能

- 路由切换时间：≤500ms
- 懒加载：页面组件按需加载，减少首屏体积
- 路由缓存：支持keep-alive缓存页面状态

#### 3.1.4 输入项

| 输入项 | 类型 | 说明 |
|-------|------|------|
| URL路径 | string | 用户访问的URL |
| 路由参数 | object | 动态路由参数，如:id |
| 查询参数 | object | URL查询参数 |
| 导航方式 | enum | push/replace/go等 |

#### 3.1.5 输出项

| 输出项 | 类型 | 说明 |
|-------|------|------|
| 页面组件 | Component | 匹配到的页面组件实例 |
| 路由信息 | Route | 当前路由信息对象 |
| 导航结果 | boolean/Error | 导航成功或失败 |

#### 3.1.6 算法

**路由匹配算法**：
1. 按路由定义顺序依次匹配
2. 优先匹配精确路径，再匹配动态路径
3. 通配符路由（:pathMatch(.*)*）最后匹配
4. 匹配成功后执行导航守卫
5. 守卫通过后渲染对应组件

**导航守卫执行顺序**：
1. 失活组件的beforeRouteLeave
2. 全局beforeEach守卫
3. 重用组件的beforeRouteUpdate
4. 路由配置的beforeEnter
5. 激活组件的beforeRouteEnter
6. 全局beforeResolve守卫
7. 导航确认
8. 全局afterEach钩子

#### 3.1.7 流程逻辑

```
用户输入URL或点击链接
    ↓
路由开始解析
    ↓
触发beforeRouteLeave守卫（离开的组件）
    ↓
触发全局beforeEach守卫
    ↓
检查是否需要认证
    ├─ 需要认证且未登录 → 重定向到登录
    └─ 不需要或已登录 → 继续
    ↓
匹配路由配置
    ↓
触发路由beforeEnter守卫
    ↓
异步加载页面组件（懒加载）
    ↓
触发beforeRouteEnter守卫（进入的组件）
    ↓
触发全局beforeResolve守卫
    ↓
导航确认
    ↓
渲染页面组件
    ↓
触发全局afterEach钩子
```

#### 3.1.8 接口

**对外接口**：
- `createRouter()`：创建路由实例
- `router.push()`：编程式导航
- `router.replace()`：替换式导航
- `router.go()`：历史记录导航
- `router.beforeEach()`：全局前置守卫
- `router.afterEach()`：全局后置钩子

**内部接口**：
- 与页面组件：通过props传递路由参数
- 与状态管理：通过Pinia获取认证状态
- 与API服务：通过服务层获取数据

#### 3.1.9 存储分配

- 路由配置：内存存储，应用启动时加载
- 路由历史：内存存储，基于浏览器History API
- 路由缓存：可选，通过keep-alive缓存组件实例

#### 3.1.10 注释设计

- 每个路由配置添加注释说明页面功能
- 导航守卫添加注释说明守卫逻辑
- 重要路由参数添加类型注释

#### 3.1.11 限制条件

- 路由路径必须以/开头
- 动态路由参数名必须合法
- 通配符路由只能放在最后
- 导航守卫中不能执行耗时操作

#### 3.1.12 测试计划

1. **单元测试**：
   - 测试路由配置的正确性
   - 测试路由匹配逻辑
   - 测试导航守卫逻辑

2. **集成测试**：
   - 测试页面间导航流程
   - 测试路由参数传递
   - 测试权限控制

3. **E2E测试**：
   - 测试完整用户导航路径
   - 测试浏览器前进后退

#### 3.1.13 尚未解决的问题

- 路由级别的代码分割优化
- 路由过渡动画的性能优化

### 3.2 API服务模块（services/apiService.ts）

#### 3.2.1 程序描述

API服务模块是前端与后端通信的核心模块，封装了所有后端API调用，提供统一的接口和类型定义。基于原生fetch API封装，支持请求拦截、响应拦截、错误处理、本地数据降级等功能。

#### 3.2.2 功能

1. 封装所有后端API调用
2. 提供TypeScript类型定义
3. 支持请求参数序列化
4. 支持响应数据解析
5. 统一错误处理
6. 支持本地数据降级方案
7. 支持请求超时控制
8. 支持数据缓存

#### 3.2.3 性能

- 请求超时：默认5秒
- 缓存策略：内存缓存，减少重复请求
- 并行请求：支持Promise.all并行请求
- 数据降级：后端不可用时使用本地数据

#### 3.2.4 输入项

| 输入项 | 类型 | 说明 |
|-------|------|------|
| textId | string | 课文ID |
| page | number | 页码 |
| pageSize | number | 每页数量 |
| textIds | string[] | 课文ID数组（批量） |
| studentId | string | 学生ID |
| answers | object | 答题答案 |

#### 3.2.5 输出项

| 输出项 | 类型 | 说明 |
|-------|------|------|
| TextBasicInfo | interface | 课文基础信息 |
| WordItem[] | interface | 字词注释列表 |
| MultiRoleReadingData | interface | 多角色朗读数据 |
| Level1QuizData | interface | 一级测验数据 |
| Level2DialogData | interface | 二级对话数据 |
| Level2QuizData | interface | 二级测验数据 |
| Level3ScenarioText | interface | 三级情景文本 |
| Level3AdaptiveQuiz | interface | 三级自适应测验 |
| ApiResponse | interface | 统一响应格式 |

#### 3.2.6 算法

**请求封装算法**：
1. 构建请求URL和参数
2. 设置请求头（Content-Type等）
3. 发起fetch请求
4. 超时控制（AbortController）
5. 解析响应数据
6. 错误处理和重试
7. 返回格式化的响应

**本地降级算法**：
1. 尝试从后端API获取数据
2. 如果失败或超时，尝试加载本地JSON文件
3. 本地文件路径映射：typeMap转换
4. 返回本地数据或抛出错误

#### 3.2.7 流程逻辑

```
调用API函数
    ↓
检查缓存中是否有数据
    ├─ 有缓存且未过期 → 直接返回缓存数据
    └─ 无缓存或已过期 → 继续
    ↓
构建请求配置（URL、方法、参数、头）
    ↓
创建AbortController（超时控制）
    ↓
发起fetch请求
    ↓
等待响应或超时
    ├─ 超时 → 抛出超时错误，尝试降级
    └─ 正常响应 → 继续
    ↓
检查响应状态
    ├─ 成功（200）→ 解析JSON
    ├─ 客户端错误（4xx）→ 抛出业务错误
    └─ 服务端错误（5xx）→ 尝试降级
    ↓
解析响应数据
    ↓
数据缓存（可选）
    ↓
返回格式化的响应数据
```

#### 3.2.8 接口

**API函数接口**：
- `getTextBasicInfo(textId: string): Promise<ApiResponse<TextBasicInfo>>`
- `getWordList(textId: string): Promise<ApiResponse<WordItem[]>>`
- `getMultiRoleReading(textId: string): Promise<ApiResponse<MultiRoleReadingData>>`
- `getLevel1Quiz(textId: string): Promise<ApiResponse<Level1QuizData>>`
- `getLevel2Dialog(textId: string): Promise<ApiResponse<Level2DialogData>>`
- `getLevel2Quiz(textId: string): Promise<ApiResponse<Level2QuizData>>`
- `getLevel3ScenarioText(textId: string): Promise<ApiResponse<Level3ScenarioText>>`
- `getLevel3AdaptiveQuiz(textId: string): Promise<ApiResponse<Level3AdaptiveQuiz>>`
- `getTextList(page?, pageSize?): Promise<ApiResponse<{total, texts}>>`
- `getTextBatch(textIds: string[]): Promise<ApiResponse<...>>`
- `getLocalData<T>(dataType, textId, timeout?): Promise<T>`

**类型定义接口**：
- TextBasicInfo
- WordItem
- MultiRoleReadingData
- Level1QuizData
- Level2DialogData
- Level2QuizData
- Level3ScenarioText
- Level3AdaptiveQuiz
- ApiResponse<T>

#### 3.2.9 存储分配

### 3.3 字词注释组件（WordList.vue）

#### 3.3.1 程序描述

WordList组件是一级学习阶段的核心组件，用于展示课文中的字词注释。支持点击字词查看详细释义，包含基本释义、近义辨析、追问问题等内容。

#### 3.3.2 功能

1. 展示字词列表
2. 点击字词展开详细注释
3. 显示基本释义
4. 显示近义辨析（可选）
5. 显示追问问题（可选）
6. 支持并行加载数据
7. 错误处理和重试

#### 3.3.3 性能

- 数据加载时间：≤1秒
- 切换字词响应：即时
- 支持大数据量：虚拟滚动（可选）

#### 3.3.4 输入项

| 输入项 | 类型 | 说明 |
|-------|------|------|
| wenId | string | 文言文ID |
| wordListBaseUrl | string | 字词列表数据URL |
| basicInfoBaseUrl | string | 基础信息URL |

#### 3.3.5 输出项

- 渲染字词注释UI
- 触发父组件事件（可选）

#### 3.3.6 算法

**数据加载算法**：
1. 并行请求word_list和text_basic_info数据
2. 使用Promise.all同时发起两个请求
3. 设置超时控制（5秒）
4. 解析JSON数据
5. 处理数据格式
6. 渲染到页面

**字词展开算法**：
1. 点击字词时切换展开状态
2. 记录当前展开的字词索引
3. 动画过渡展开/收起

#### 3.3.7 流程逻辑

```
组件挂载

    └─ 否 → 显示错误，提供重试
    ↓

    ↓
```

#### 4.3.6 算法

**答案比较算法**：
1. 判断答案类型（单选/多选）
2. 单选：统一转换为字符串后比较
   - `String(correctAnswer) === String(userAnswer)`
3. 多选：排序后转换为JSON字符串比较
   - `sortedCorrect = [...correctAnswer].map(String).sort()`
   - `sortedUser = [...userAnswer].map(String).sort()`
   - `JSON.stringify(sortedCorrect) === JSON.stringify(sortedUser)`
4. 正确得100分，错误得0分

**尝试次数统计算法**：
1. 查询该学生该题目的历史记录数
2. `attemptNumber = count + 1`
3. 每次提交都保留记录，不覆盖

**得分统计算法**：
1. `totalScore = sum(所有题目得分)`
2. `correctCount = sum(isCorrect === 1)`
3. `wrongCount = totalQuestions - correctCount`
4. `avgScore = round(totalScore / totalQuestions)`

```
    ↓
判断错误类型
    ├─ 业务错误（有statusCode）→ 使用自定义状态码

    ├─ 数据库错误 → 状态码500

构建错误响应对象
    ↓
```

### 6.2 日志工具模块（utils/logger.js）

#### 6.2.1 程序描述

logger是日志工具模块，提供统一的日志记录功能，支持不同日志级别、多种输出格式、文件日志等。

#### 6.2.2 功能

1. 提供不同级别的日志方法（info, warn, error, debug）
2. 支持控制台输出和文件输出
3. 支持JSON格式和文本格式
4. 日志文件轮转（按大小和数量）
5. 操作日志记录
6. 日志级别控制

#### 6.2.3 性能

- 日志写入：异步，不阻塞主流程
- 日志格式化：≤1ms
- 文件轮转：按配置自动执行

#### 6.2.4 输入项

| 输入项 | 类型 | 说明 |
|-------|------|------|
| message | string | 日志消息 |
| data | object | 附加数据（可选） |
| level | string | 日志级别 |

#### 6.2.5 输出项

- 控制台日志输出
- 文件日志输出
- 日志格式：文本或JSON

#### 6.2.6 算法

**日志级别判断算法**：
1. 比较当前日志级别和配置级别
2. 当前级别 >= 配置级别才输出
3. 级别顺序：DEBUG < INFO < WARN < ERROR

**日志格式化算法**：
1. 添加时间戳
2. 添加日志级别
3. 格式化附加数据
4. 文本格式：拼接字符串
5. JSON格式：序列化为JSON

#### 6.2.7 流程逻辑

```
调用日志方法（info/warn/error等）
    ↓
检查日志级别
    ├─ 级别不够 → 直接返回
    └─ 级别足够 → 继续
    ↓
格式化日志内容
    ↓
输出到控制台
    ↓
配置了文件输出？
    ├─ 是 → 写入日志文件
    └─ 否 → 结束
    ↓
检查文件大小
    ├─ 超过限制 → 轮转文件
    └─ 未超过 → 结束
```

#### 6.2.8 接口

**对外接口**：
- `info(message, data?)`：信息日志
- `warn(message, data?)`：警告日志
- `error(message, data?)`：错误日志
- `debug(message, data?)`：调试日志
- `logOperation(operation, data?)`：操作日志

**配置依赖**：
- config.logger.level：日志级别
- config.logger.format：日志格式
- config.logger.outputDir：输出目录
- config.logger.maxFileSize：最大文件大小
- config.logger.maxFiles：最大文件数

#### 6.2.9 存储分配

- 日志实例：内存存储，单例
- 日志缓冲区：内存存储（如使用winston）
- 日志文件：磁盘存储

#### 6.2.10 注释设计

- 模块功能说明
- 每个方法JSDoc注释

#### 6.2.11 限制条件

# Python脚本模块拆分说明文档

## 一、重构背景

原有的Excel转JSON工具是一个单一的Python文件，随着业务需求的增长，代码变得越来越复杂，难以维护和扩展。为了提升代码结构的清晰度、可维护性和可扩展性，我们将原有的1个Python文件拆分为多个功能模块。

## 三、模块职责说明

### 3.1 工具层（utils/）

| 模块 | 职责 | 核心功能 |
|------|------|---------|
| `excel_reader.py` | Excel文件读取 | 打开工作簿、读取表头、提取数据行 |
| `json_writer.py` | JSON文件输出 | 保存JSON、格式化输出、按组保存 |
| `data_transformer.py` | 通用数据转换 | 字母转索引、分组、验证、空值处理 |

### 3.2 转换器层（transformers/）

| 模块 | 对应工作表 | 数据结构 |
|------|-----------|---------|
| `text_basic_info.py` | `text_basic_info` | 课文基础信息（标题、作者、原文等） |
| `level1_word_list.py` | `level1_word_list` | 字词注释（词语、释义、辨析等） |
| `level1_multi_role_reading.py` | `level1_multi_role_reading` | 多角色朗读（角色、台词、时间范围） |
| `level1_quiz.py` | `level1_quiz` | 一级测验题目（单选、多选、判断） |
| `level2_dialog_quiz.py` | `level2_dialog_quiz` | 二级对话测验（对话文本、分支选项） |
| `level3_adaptive_quiz.py` | `level3_adaptive_quiz` | 三级自适应测验（难度分级、知识点） |
| `culture_cards.py` | `culture_cards` | 文化卡片（标题、内容、配图） |

## 六、设计原则

### 6.1 单一职责原则

每个模块只负责一项功能：
- `excel_reader.py`: 只处理Excel读取
- `json_writer.py`: 只处理JSON输出
- `data_transformer.py`: 只处理通用数据转换
- 每个转换器模块只处理对应工作表的数据

### 6.2 依赖倒置原则

高层模块（`generate_all_json.py`）不依赖低层模块的具体实现，而是通过映射表（`TRANSFORM_MAP`、`GROUP_MAP`）进行间接调用。

### 6.3 开闭原则

扩展新的工作表处理只需：
1. 在 `transformers/` 目录下新建转换器模块
2. 在 `transformers/__init__.py` 的映射表中注册

无需修改现有代码。

## 十一、Level2（StepTwo）模块详解

### 11.1 组件架构

Level2模块主要由以下组件构成：

| 组件文件 | 功能描述 |
|---------|---------|
| `src/views/StepTwoView.vue` | Level2主页面，负责数据加载和页面布局 |
| `src/components/BlockRenderer.vue` | 动态渲染器，根据block.type渲染对应组件 |
| `src/components/DialogueCard.vue` | 对话卡片组件，展示对话内容 |
| `src/components/AdaptQuiz.vue` | 自适应测验组件，处理题目交互 |

### 11.2 数据流转流程

1. **数据加载**：`StepTwoView`通过`useDataLoader`从`/data/pages_level2_dialog_quiz/{wenId}.json`加载页面配置
2. **动态渲染**：`BlockRenderer`根据配置中的`blocks`数组，依次渲染dialogue和quiz类型的块
3. **交互处理**：用户答题时，`AdaptQuiz`触发`quiz-answer`事件，`StepTwoView`处理并更新进度
4. **导航控制**：完成所有题目后，`BackContinue`组件显示"继续"按钮，允许进入下一关

### 11.4 Python转换器（level2_dialog_quiz.py）

该转换器负责：
- 从Excel工作表`level2_dialog_quiz`读取数据
- 根据`question_number`字段区分dialogue和quiz类型
- 自动生成`question_id`（格式：`{text_id}_B{question_number}`）
- 按`text_id`分组并保存到`/public/data/pages_level2_dialog_quiz/`目录

## 十二、Level3（StepThree）模块详解

### 12.1 组件架构

Level3模块主要由以下组件构成：

| 组件文件 | 功能描述 |
|---------|---------|
| `src/views/StepThreeView.vue` | Level3主页面，情景测验模式 |
| `src/components/QuizCard.vue` | 测验卡片组件，处理题目交互和答案显示 |
| `src/components/BackContinue.vue` | 导航按钮组件 |

### 12.2 数据流转流程

1. **数据加载**：`StepThreeView`通过`useDataLoader`从`/data/pages_level3_adaptive_quiz/{wenId}.json`加载测验数据
2. **题目渲染**：按顺序显示`items`数组中的题目，情景文本显示在第一题上方
3. **答案提交**：用户选择答案后，`QuizCard`触发提交事件，记录答案并显示解析
4. **进度管理**：使用`useQuizProgress`管理答题进度，完成后显示"继续"按钮

### 12.3 JSON数据结构（Level3）

```json
{
  "pageId": "WEN_01",
  "items": [
    {
      "text": "情景文本...",
      "quiz": {
        "question_id": "WEN_01_C1",
        "module": "C",
        "question_type": "radio",
        "question_text": "题目文本...",
        "options": ["选项A", "选项B", "选项C", "选项D"],
        "correct_answer": 1,
        "explanation": "答案解析...",
        "difficulty": "L1"
      }
    }
  ]
}
```

### 12.4 Python转换器（level3_adaptive_quiz.py）

该转换器负责：
- 从Excel工作表`level3_adaptive_quiz`读取数据
- 处理重复列名问题（Excel中有两个`question_text`列）
- 第一个`question_text`作为选项A，第二个作为题目文本（通过特殊逻辑处理）
- 使用`convert_letter_to_index`将字母答案（A/B/C/D）转换为数字索引
- 自动生成`question_id`（格式：`{text_id}_C{序号}`）
- 按`text_id`分组并保存到`/public/data/pages_level3_adaptive_quiz/`目录

## 十三、关键技术发现

### 13.1 数据驱动的页面设计

项目采用了数据驱动的页面设计理念：
- Level2使用Block模式，通过配置动态组合不同类型的内容块
- Level3使用Item模式，按顺序展示情景测验题目
- 这种设计使得页面内容完全由JSON配置控制，无需修改代码即可更新内容

### 13.2 统一的组合式函数

项目使用以下核心组合式函数：
- `useDataLoader`：统一的数据加载接口，处理加载状态、错误处理
- `useNavigation`：统一的页面导航逻辑
- `useQuizProgress`：统一的测验进度管理

### 13.3 Excel列名处理的特殊问题

在Level3的Excel中发现重复列名的问题：
- 两个列都命名为`question_text`，第一个是选项A，第二个是题目文本
- 由于Python字典键的唯一性，需要特殊逻辑来处理这种情况
- 当前解决方案：通过检查scenario_text中是否包含问号来决定如何设置question_text

## 1. 502 Bad Gateway 错误

### 错误信息

```
[error] 获取数据失败: ApiError: 请求失败: 502
```

### 错误原因

502 错误表示前端向网关（如 Nginx、Vite 代理）发送请求，但网关无法从上游服务器（后端服务）获取有效响应。常见原因：

| 原因 | 说明 |
|------|------|
| 后端服务未启动 | Node.js 服务器未运行或已崩溃 |
| 端口占用 | 后端服务端口被其他进程占用 |
| 网络配置问题 | 防火墙阻止了端口访问 |
| 代理配置错误 | Vite 代理配置指向错误地址 |
| 服务启动超时 | 后端服务启动时间过长 |

### 排查步骤

```
1. 检查后端服务状态
   └── 查看终端输出，确认服务是否正常运行
   
2. 验证端口可达性
   └── 访问 http://localhost:3000 确认后端服务可用
   
3. 检查代理配置
   └── 确认 vite.config.ts 中 proxy 配置正确
   
4. 查看服务器日志
   └── 检查后端服务器的错误日志
```

### 解决方案

1. **重新启动后端服务**：
   ```bash
   cd backend
   node server.js
   ```

2. **检查端口占用**：
   ```bash
   # Windows
   netstat -ano | findstr :3000
   
   # 杀死占用进程
   taskkill /PID <PID> /F
   ```

3. **验证 Vite 代理配置**：
   ```typescript
   // vite.config.ts
   server: {
     proxy: {
       '/api': {
         target: 'http://localhost:3000',
         changeOrigin: true,
         secure: false,
       },
     },
   }
   ```

## 2. 答题数据未保存到数据库

### 错误现象

```
控制台显示答案提交成功，但数据库 answer_records 表为空
```

### 错误原因

前端组件使用 `useQuizProgress` 管理答题进度，但该 composable 只维护前端状态，没有调用后端 API 将数据保存到数据库。

### 排查步骤

```
1. 查询数据库确认数据是否存在
   └── SELECT * FROM answer_records;
   
2. 检查前端代码是否调用了 submitAnswers API
   └── 搜索 useQuizProgress.ts 中是否有 submitAnswers 调用
   
3. 检查网络请求日志
   └── 浏览器开发者工具 → Network → 查找 /api/submit 请求
```

### 解决方案

在 `useQuizProgress.ts` 中添加后端提交逻辑：

```typescript
// src/composables/useQuizProgress.ts
async function submitAnswersToBackend(): Promise<void> {
  // 1. 获取学生ID
  const studentStore = useStudentStore()
  const studentId = studentStore.studentId || studentStore.displayId || ''
  
  if (!studentId) {
    console.warn('[useQuizProgress] 未登录，跳过后端提交')
    return
  }

  // 2. 构建提交数据
  const questions = answers.value.map((ans) => ({
    id: `question_${ans.questionIndex}`,
    correctAnswer: ans.isCorrect !== undefined && ans.isCorrect ? ans.answer : 'unknown',
  }))

  const answerMap: Record<string, any> = {}
  answers.value.forEach((ans) => {
    answerMap[`question_${ans.questionIndex}`] = ans.answer
  })

  // 3. 调用 API
  await submitAnswers(
    { answers: answerMap, questions },
    completionKeyPrefix,
    studentId
  )
}

// 在 handleSubmit 中，当所有题目完成时调用
if (isCompleted.value) {
  saveCompletionRecord()
  await submitAnswersToBackend()
}
```

## 3. SQL 查询字段映射错误

### 错误现象

```
前端页面显示"查询失败"或数据为空
后端日志显示 SQL 执行错误
```

### 错误原因

| 原因 | 说明 |
|------|------|
| 字段不存在 | SELECT 语句引用了表中不存在的字段 |
| 字段名冲突 | 多表连接时字段名重复且未使用别名 |
| 表名错误 | FROM 子句中表名拼写错误 |
| WHERE 条件错误 | 条件表达式语法错误或字段类型不匹配 |

### 排查步骤

```
1. 查看后端错误日志
   └── 定位具体的 SQL 错误信息
   
2. 验证表结构
   └── SHOW COLUMNS FROM <table_name>;
   
3. 检查 SQL 语句
   └── 确认所有字段名和表名正确
```

### 解决方案

```sql
-- 错误示例：引用不存在的字段
SELECT id, title, non_existent_column FROM questions;

-- 正确示例：使用存在的字段
SELECT id, content, type FROM questions;

-- 多表连接时使用别名
SELECT 
  q.id AS question_id,
  q.content,
  epq.score AS paper_score
FROM questions q
JOIN exam_paper_question epq ON q.id = epq.question_id;
```

## 4. TypeScript 类型错误

### 错误现象

```
Type 'string | undefined' is not assignable to type 'string'
Type 'null' is not assignable to type 'string'
```

### 错误原因

| 原因 | 说明 |
|------|------|
| 可选属性未处理 | 访问可能为 undefined 的属性 |
| 空值未检查 | 变量可能为 null 但未做空值检查 |
| 类型推断错误 | TypeScript 无法正确推断类型 |

### 解决方案

```typescript
// 错误示例
const name: string = getUserName(); // getUserName 可能返回 undefined

// 正确示例：使用可选链和空值合并
const name: string = getUserName() ?? '未知用户';

// 类型守卫
function processUser(user: User | null) {
  if (!user) {
    throw new Error('用户不存在');
  }
  // user 现在被推断为非空
  console.log(user.name);
}

// 类型断言（谨慎使用）
const value = getValue() as string;
```

## 5. 前端组件数据流转问题

### 错误现象

```
组件之间数据传递失败
状态更新后界面未刷新
数据格式不匹配
```

### 错误原因

| 原因 | 说明 |
|------|------|
| Props 命名不一致 | 父组件传入的 props 名称与子组件期望的不匹配 |
| 事件名称错误 | emit 事件名称与监听事件名称不一致 |
| 响应式丢失 | 直接修改对象属性而非使用响应式 API |
| 异步数据未处理 | 组件挂载时数据尚未加载完成 |

### 解决方案

```typescript
// Props 定义
interface Props {
  wenId: string
  autoLoad?: boolean
}

const props = withDefaults(defineProps<Props>(), {
  autoLoad: true,
})

// 事件定义
const emit = defineEmits<{
  (e: 'load-success', data: unknown): void
  (e: 'load-error', error: string): void
}>()

// 响应式更新
const data = ref(null)
data.value = newData // 正确

const obj = reactive({ name: '' })
obj.name = '新名称' // 正确
```

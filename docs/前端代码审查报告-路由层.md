#### ROUTE-003: to.name可能为undefined

**问题描述**:
```typescript
if (requiresAuth(to.name as string) && !authStore.isLoggedIn)
```
`to.name`可能为`undefined`（如404路由），直接转为string会得到`"undefined"`字符串

**修复建议**:
```typescript
const routeName = to.name as string | undefined
if (routeName && requiresAuth(routeName) && !authStore.isLoggedIn)
```

**预估Bug率**: <5%（可安全修复）

---

#### ROUTE-004: protectedRoutes与路由配置重复

**问题描述**:
```typescript
const protectedRoutes = ['stepone', 'steptwo', 'stepthree']
```
与路由配置中的`meta: { requiresAuth: true }`重复，维护成本高

**修复建议**:
```typescript
function requiresAuth(to: RouteLocationNormalized): boolean {
  return !!to.meta.requiresAuth
}
```

**预估Bug率**: <5%（可安全修复）

---

#### ROUTE-005: 登录验证逻辑不完整

**问题描述**:
当前实现只是标记需要登录，没有真正拦截未登录用户

**修复建议**:
```typescript
if (requiresAuth(to) && !authStore.isLoggedIn) {
  // 记录目标路径，登录成功后跳转
  to.meta.redirectAfterLogin = true
  next({ name: 'home' }) // 跳转到首页或登录页
  return
}
```

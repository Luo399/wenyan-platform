## 3. 数据库设计

### 3.1 表结构

#### students 表（学生表）

```sql
CREATE TABLE students (
  student_id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  created_at TEXT NOT NULL DEFAULT CURRENT_TIMESTAMP
)
```

| 字段名 | 类型 | 说明 |
|--------|------|------|
| student_id | TEXT | 学号（主键） |
| name | TEXT | 学生姓名 |
| created_at | TEXT | 创建时间 |

#### answer_records 表（答题记录表）

```sql
CREATE TABLE answer_records (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  wen_id TEXT NOT NULL,
  student_id TEXT NOT NULL,
  question_id TEXT NOT NULL,
  user_answer TEXT NOT NULL,
  correct_answer TEXT,
  is_correct INTEGER DEFAULT 0,
  score INTEGER DEFAULT 0,
  submitted_at TEXT NOT NULL,
  attempt_number INTEGER DEFAULT 1
)
```

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | INTEGER | 自增主键 |
| wen_id | TEXT | 文言文ID |
| student_id | TEXT | 学号 |
| question_id | TEXT | 题目ID |
| user_answer | TEXT | 用户答案 |
| correct_answer | TEXT | 正确答案 |
| is_correct | INTEGER | 是否正确 (0/1) |
| score | INTEGER | 得分 |
| submitted_at | TEXT | 提交时间 |
| attempt_number | INTEGER | 答题次数 |

### 3.2 索引

```sql
CREATE INDEX idx_wen_id ON answer_records(wen_id)
CREATE INDEX idx_student_id ON answer_records(student_id)
CREATE INDEX idx_wen_student ON answer_records(wen_id, student_id)
```

## 4. 关键问题修复记录

### 4.1 答题记录覆盖问题

**问题描述**: 原设计使用 `(wen_id, student_id, question_id)` 作为主键，导致第二次答题会覆盖第一次的记录。

**修复方案**:
1. 将主键改为自增 `id`
2. 添加 `attempt_number` 字段记录答题次数
3. 将 `INSERT OR REPLACE` 改为 `INSERT`，保留历史记录

**修复日期**: 2026-05-29

### 4.2 学号格式限制问题

**问题描述**: 原设计要求学号必须是4位数字，与实际需求不符。

**修复方案**:
1. 移除4位数字长度限制
2. 仅验证学号为非空数字

**修复日期**: 2026-05-28

# 浏览器兼容性测试清单

## 测试环境
- Windows 7 操作系统 + Internet Explorer 11 浏览器（如项目需求明确要求支持）
- Windows 10 操作系统 + Microsoft Edge 浏览器（基于 Chromium 内核的最新稳定版本）
- Windows 10 操作系统 + 360安全浏览器（需分别测试极速模式与兼容模式）
- Windows 10 操作系统 + QQ浏览器（最新稳定版本）
- 移动端设备（需包含主流操作系统及版本：iOS 14+、Android 10+，并覆盖至少3种不同屏幕尺寸）

## 核心功能检查项

### 1. 基础加载
- [ ] 所有已定义页面路由均可通过直接访问或导航方式正常加载，无空白页面或持续加载现象
- [ ] 浏览器开发者控制台无 JavaScript 错误（红色报错），允许存在已提前声明的 IE 浏览器不支持特性的警告日志
- [ ] 通用状态组件（BaseLoader/BaseError/BaseEmpty）在对应场景下能够正确出现和消失，动画效果符合设计规范
- [ ] 404错误页面在访问不存在路由时正确显示，包含返回首页功能

### 2. 字词学习
- [ ] 课文内容完整正确显示，HTML特殊字符已正确转义，无乱码、重叠或截断现象
- [ ] 鼠标悬浮于词汇时，Tooltip提示框在500ms内正常弹出，位置不超出视窗范围
- [ ] Tooltip内注释文字完整展示，无水平或垂直方向溢出，不遮挡页面其他关键内容

### 3. 音频/视频播放
- [ ] 页面加载完成后，音频应按设计要求自动播放（针对Chrome/Edge等需要用户交互才能自动播放的浏览器，需在测试报告中记录实际行为差异）
- [ ] 音频播放过程中，对应文本段落随播放进度准确高亮，段落切换无延迟或错位
- [ ] 视频内容能够自动播放，无 ERR_ABORTED 或其他加载错误，首屏加载时间不超过3秒
- [ ] 音视频播放控件（播放/暂停按钮、进度条、音量控制）功能正常可用，交互反馈及时

### 4. 交互与导航
- [ ] 页面底部 Back/Continue 按钮点击响应时间不超过300ms，能够准确跳转到预期页面
- [ ] 所有可交互按钮元素（包括自定义按钮组件）点击区域完整，样式在不同状态（默认/悬停/点击/禁用）下显示正常，无错位或变形

### 5. 响应式/UI
- [ ] 页面在1024px宽度（平板横向）和1920px宽度（桌面端）下布局保持一致，关键内容无重叠或溢出
- [ ] 页面字体大小、颜色值、元素间距符合设计规范，无明显错位或显示异常
- [ ] 所有图片、图标资源加载完整，无破损或错误显示，SVG图标缩放无失真

> 注：后续可根据新增业务模块或功能点，在对应分类下增删检查项，确保保持清单结构清晰。

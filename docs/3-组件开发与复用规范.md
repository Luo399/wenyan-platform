# 3-组件开发与复用规范

## 1. 组件分类

### 1.1 组件类型

| 类型 | 职责 | 项目示例 |
|-----|------|---------|
| **业务组件** | 核心业务功能 | `WordList`, `MultiRoleReading`, `VideoPlayer` |
| **UI组件** | 通用界面元素 | `BackContinue`, `ErrorDisplay` |
| **布局组件** | 页面结构布局 | `PoetryMenu` |
| **表单组件** | 用户输入交互 | `StudentLogin`, `StudentDisplay` |

### 1.2 组件命名规范

- **文件名**: PascalCase，如 `WordList.vue`, `MultiRoleReading.vue`
- **组件名**: PascalCase，与文件名一致
- **props**: camelCase，如 `wenId`, `autoLoad`
- **events**: kebab-case，如 `@load-success`, `@segment-change`

## 2. Props设计原则

### 2.1 Props定义规范（`WordList.vue`示例）

```typescript
// src/components/WordList.vue
interface Props {
  wenId: string                    // 必填：课文ID
  autoLoad?: boolean               // 可选：是否自动加载
  wordListBaseUrl?: string         // 可选：字词列表数据路径
  basicInfoBaseUrl?: string        // 可选：课文基础信息路径
}

const props = withDefaults(defineProps<Props>(), {
  autoLoad: true,
  wordListBaseUrl: '/data/word_list/',
  basicInfoBaseUrl: '/data/text_basic_info/'
})
```

### 2.2 Props命名规则

| 类型 | 命名示例 | 说明 |
|-----|---------|-----|
| 布尔型 | `autoLoad`, `cacheEnabled`, `isPlaying` | 使用 `auto`/`is`/`has`/`should` 前缀 |
| 字符串 | `wenId`, `resourcePath`, `backText` | 描述性名称 |
| 数字 | `timeout`, `currentTime`, `duration` | 明确单位（毫秒/秒） |
| 对象/数组 | `segments`, `wordList`, `style` | 复数形式 |

## 3. 事件设计规范

### 3.1 事件命名

- 使用 kebab-case（如 `load-success`, `segment-change`）
- 过去式表示完成（`load-success`, `load-error`, `play-ended`）
- 现在式表示进行中（`playing`, `loading`）

### 3.2 事件定义（`MultiRoleReading.vue`示例）

```typescript

  (e: 'segment-change', index: number): void

// 触发事件示例
emit('load-success', multiRoleData)
emit('segment-change', currentSegmentIndex.value)
```

## 4. 状态管理规范

### 4.1 组件内部状态（`WordList.vue`示例）

```typescript
// src/components/WordList.vue
const loading = ref(true)           // 加载状态
const error = ref<string | null>(null) // 错误信息
const wordList = ref<WordItem[]>([]) // 字词数据
const basicInfo = ref<TextBasicInfo | null>(null) // 课文基础信息
const showTooltip = ref(false)      // 注释弹窗显示状态
const currentAnnotation = ref('')   // 当前注释内容
const tooltipPosition = ref({ x: 0, y: 0 }) // 弹窗位置
```

### 4.2 状态命名规则

- **布尔状态**: 使用 `is` 前缀（`isPlaying`, `isLoaded`, `showTooltip`）
- **完成状态**: 使用过去分词（`loaded`, `initialized`）
- **动作状态**: 使用主动语态动词（`loading`, `processing`）
- **数据状态**: 使用名词（`wordList`, `basicInfo`, `segments`）

### 4.3 状态管理最佳实践

1. **最小化状态**: 只保留组件必需的状态
2. **状态初始化**: 所有状态应有合理的初始值
3. **状态分离**: 将UI状态与业务状态分离

## 7. 错误处理规范

### 7.1 错误状态处理

```typescript
// src/components/WordList.vue - loadData函数
try {
  // 请求数据
  const [wordListResponse, basicInfoResponse] = await Promise.all([
    fetch(wordListUrl, { signal: abortController.signal }),
    fetch(basicInfoUrl, { signal: abortController.signal })
  ])
  
  // 检查HTTP状态
  if (!wordListResponse.ok) {
    throw new Error(`word_list 加载失败: HTTP ${wordListResponse.status}`)
  }
  
  // 解析数据
  const wordListBuffer = await wordListResponse.arrayBuffer()
  const wordListText = new TextDecoder('utf-8').decode(wordListBuffer)
  const wordListData: WordItem[] = JSON.parse(wordListText)
  
  // 数据验证
  if (!Array.isArray(wordListData)) {
    throw new Error('word_list 数据格式错误：应为数组')
  }
  
} catch (err) {
  if (err instanceof DOMException && err.name === 'AbortError') {
    error.value = '加载超时'
    return
  }
  
  const errorMsg = err instanceof Error ? err.message : '加载失败'
  console.error('[WordList] ❌ 数据加载失败:', errorMsg)
  
  if (errorMsg.includes('404')) {
    error.value = '【404正在加班加点中】'
  } else {
    error.value = errorMsg
  }
}
```

## 8. 组件复用原则

### 8.1 可复用组件特征

| 特征 | 说明 | 示例 |
|-----|------|-----|
| **无外部依赖** | 不依赖特定上下文 | `BackContinue`, `ErrorDisplay` |
| **明确输入输出** | 清晰的props和events | `WordList` (props: wenId, events: 无) |
| **自包含状态** | 状态管理在组件内部 | `MultiRoleReading` |
| **良好错误处理** | 完善的错误处理机制 | 所有业务组件 |

### 8.2 不可复用场景

- **高度业务耦合**：如特定页面的定制化组件
- **依赖特定上下文**：如依赖路由参数的组件
- **一次性使用**：如单页面独有的布局组件

### 8.3 复用模式示例

**高阶组件模式**（用于跨组件共享逻辑）：

```typescript
// 可创建 src/composables/useDataLoader.ts
export function useDataLoader<T>(loadFn: () => Promise<T>) {
  const loading = ref(false)
  const error = ref<string | null>(null)
  const data = ref<T | null>(null)

  async function load() {
    loading.value = true
    error.value = null
    
    try {
      data.value = await loadFn()
    } catch (err) {
      error.value = err instanceof Error ? err.message : '加载失败'
    } finally {
      loading.value = false
    }
  }

  return { loading, error, data, load }
}

// 在组件中使用
// const { loading, error, data, load } = useDataLoader(() => fetchWordList(wenId))
```

## 10. 数据加载策略规范

### 10.1 数据来源与useDataLoader使用判定标准

| 数据来源类型 | 典型组件示例 | 是否需要使用useDataLoader | 说明 |
|------------|------------|-------------------------|------|
| 自行fetch JSON数据 | WordList、MultiRoleReading | ✅ 需要 | 所有通过fetch API获取JSON数据的场景，必须统一使用useDataLoader进行数据加载管理，确保数据加载逻辑的一致性和可维护性 |
| 父组件通过props传入 | Question、Options、PoetryMenu | ❌ 不需要 | 通过组件props接收的数据，直接使用props传递的值，无需额外使用useDataLoader |
| 全局store获取 | StudentDisplay | ❌ 不需要 | 从全局状态管理store（如Pinia）获取的数据，直接通过store API获取，无需使用useDataLoader |
| 用户交互/表单数据 | StudentLogin | ❌ 不需要 | 表单输入、用户交互产生的临时数据，通过组件内部state管理，无需使用useDataLoader |
| 媒体元素原生事件 | VideoPlayer | ❌ 不需要 | 媒体元素（如video、audio）的加载状态和事件处理，应使用原生媒体事件（如onLoadedData、onError等），无需使用useDataLoader |

### 10.2 实施要求

1. **强制遵循**：所有新开发组件必须严格遵循上述判定标准选择数据加载方式
2. **逐步重构**：现有组件如不符合此规范，应在后续迭代中逐步重构调整
3. **完整实现**：使用useDataLoader时，必须实现完整的加载状态、错误处理和数据缓存逻辑

### 10.3 useDataLoader使用示例

```typescript
// 在组件中使用useDataLoader
import { useDataLoader } from '@/composables/useDataLoader'

const dataUrl = computed(() => `/data/word_list/${props.wenId}.json`)

const {
  loading,
  error,
  isTimeout,
  data: wordListData,
  retry
} = useDataLoader<WordItem[]>(dataUrl, {
  timeout: 10000,
  retryCount: 1,
  cacheEnabled: true
})
```

### 10.4 状态处理模板

```vue

  <div>
    <!-- 加载状态 -->

    </div>

```
